# Audio Splitter GUI

[English](./README.md)

一个轻量级的 Rust + WASAPI Windows 工具：将系统播放声音复制到额外输出线路（例如 `CABLE Input`），同时保持耳机/扬声器原始播放不断。

## 功能

- 基于 WASAPI loopback 采集系统播放声音
- 将声音复制到你选择的输出设备（虚拟声卡、硬件线路等）
- 极简 GUI：刷新设备、选择输出、启动/停止
- 输入/输出链路实时电平可视化

## 适用场景

- 将播放器声音重定向为“麦克风输入”
- 语音转录流程（ASR）
- 会议记录采集
- 通话录音/归档
- 将声音送入其他软件线路或后处理工具

## 环境要求

- Windows 10/11
- Rust 工具链（源码编译时需要）
- 如需让其他软件把它当麦克风使用，需要安装虚拟音频线驱动

## VB-Audio Virtual Cable 安装建议

`VB-Audio Virtual Cable` 不是 Windows 系统自带，需要手动安装第三方驱动。

推荐配置流程：

1. 安装 VB-CABLE（如提示重启请重启）。
2. Windows 默认播放设备继续使用你的耳机/扬声器。
3. 在本工具中将输出目标设为 `CABLE Input`，然后点击 `Start`。
4. 在语音/会议软件中将麦克风（输入）设为 `CABLE Output`。
5. 在语音输入软件中关闭类似 **“语音输入时静音”** / **“说话时自动静音”** 的选项。

## 构建与运行

```bash
cargo run --release
```

## 路由说明

- 本项目不会自动创建虚拟设备，只会使用系统中已存在的设备。
- 如果录音时耳机没声音，通常是把系统默认播放错误切到了 `CABLE Input`。将默认播放改回耳机/扬声器即可。
- 建议尽量统一使用 48 kHz，提升链路稳定性。
